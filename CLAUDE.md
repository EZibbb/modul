- **ЧЕКЛИСТ перед любой задачей.** Без чеклиста не начинать.
- **6-шаговый отчёт после каждого блока** (см. `INSTRUCTIONS.md`).
- **Верификация: 3 фазы + дыры** (внутренняя / перекрёстная / полнота / слабое место).
- **ОДИН вопрос за раз.** Без вступлений «Конечно! Сейчас я...». Таблицы вместо абзацев.
- **Не выходить за рабочую папку `c:\Users\Cax1a\Desktop\Modul comp` без разрешения.**
- **Триггеры конца сессии** («на сегодня всё», «пока», «закончили») → автоматическая процедура.
- **Аудит на дыры после каждой порции** — 4 категории (Критические / Средние / Слабые / Незакрытые), применять найденное без отдельного вопроса.
- **Согласовать драфт ДО записи в файл** — порции фич / новые разделы концепции / крупные реструктуризации.
- **Push на GitHub каждые 5 коммитов** или сразу после критического фикса.
- **Дневник LOG обязателен** — события сессии фиксирую в `SESSION_CONTEXT.md → LOG` сразу, не в конце.
- **Не предлагать закрытие сессии** — после задач предлагать конкретный следующий шаг, не «закрываемся?». Закрытие — только по триггеру.

## KNOWLEDGE GRAPH (Graphify) — экономия токенов при работе с проектом

> Проект индексируется Graphify. Граф живёт в `graphify-out/` (создаётся при первом `/graphify`).
> Цель: не перечитывать все файлы каждую сессию — сначала смотреть в граф.

**Правила:**
1. **Сначала граф, потом raw-файл.** Перед чтением `_claude/*.md`, `concept/*.md` и т.д. — сначала смотри в `graphify-out/GRAPH_REPORT.md` (сводка) или `graphify-out/graph.json` (детали).
2. **Raw-файлы читай только когда нужен точный текст** — для редактирования, цитирования, или когда граф не дал ответа.
3. **Для навигации по концепциям** — используй `graphify-out/GRAPH_REPORT.md` → секции God Nodes / Communities / Hyperedges.
4. **При изменениях в `_claude/`, `concept/`, корневых файлах** — после сессии запускаем `/graphify --update` (только пере-индексация изменённого, не весь граф заново).
5. **После каждого `/graphify`** — обязательно прогонять `python graphify-out/_apply_enhancements.py` (iOS-minimalist styling). Скрипт лежит в `_OPTIONAL/graphify-enhancements/` и копируется в `graphify-out/` при первом запуске.
6. **Для конкретного вопроса по графу** — `/graphify query "вопрос"` (BFS) или `--dfs` (трассировка пути).

Что НЕ трогать:
- `graphify-out/cache/` — кеш для `--update`, удаление приведёт к полной переиндексации
- `graphify-out/manifest.json` — diff с прошлым запуском
