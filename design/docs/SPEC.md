## 1. Обзор продукта и цели

**Modul comp** — интернет-магазин оптических трансиверов и сетевого оборудования: SFP / SFP+ / SFP28 / QSFP+ / QSFP28 / QSFP-DD / OSFP модули, DAC/AOC-кабели, патч-корды, CWDM/DWDM-мультиплексоры и готовые решения.

### Аудитория
Инженеры связи, сетевые администраторы, проектировщики, интеграторы, отделы закупок. Технически придирчивая аудитория, которая ценит:
- точность данных и плотные читаемые таблицы характеристик;
- быстрый поиск и подбор по параметрам;
- отсутствие «маркетинговой воды»;
- инженерное доверие (тесты, статусы наличия, сертификаты).

### Цели интерфейса
| Цель | Как достигается |
|---|---|
| Инженерное доверие | Бейджи «протестировано», статусы наличия, DOM-диагностика, полные datasheet |
| Скорость подбора | Экспресс-подбор, ⌘K-палитра, AI-со-пилот, подбор по портам |
| Плотность без хаоса | Зебра-таблицы, числа по правому краю, моноширинные артикулы, иерархия |
| Экономическая прозрачность | Цена + партнёрская + «экономия vs OEM» на каждой позиции |
| AI-native | Со-пилот, командная палитра, ИИ-диагностика DOM как сквозные инструменты |

### Бенчмарки
- **FS.com** — каталог трансиверов (сделано чище и современнее);
- **Linear / Vercel / Stripe Dashboard** — уровень полировки;
- **datasheet-стилистика** — моноширинные акценты, аккуратные таблицы, чёткая типографика.

## 2. Дизайн-принципы и концепция

### Концепция: «инженерная точность как интерфейс»
Приборная панель, а не витрина гаджетов. Данные — герой. Сдержанная палитра отдаёт пиксели таблицам и числам; декор минимизирован.

### Пять принципов
1. **Инженерная точность** — чистая сетка, спокойная палитра, данные = герой.
2. **Плотность без хаоса** — таблицы и фильтры плотные, но читаемые (иерархия, зебра, числа по правому краю).
3. **Технологичность, не игрушечность** — сдержанный акцент: глубокий оптический синий + циан (фотоника).
4. **Доверие** — бейджи «протестировано», статусы наличия, сертификаты как знаки качества.
5. **Адаптивность** — десктоп-first, мобильные фильтры в offcanvas, тач-зоны ≥ 44px.

### Оптический визуальный язык
Фирменный приём, отличающий продукт от «обычного shadcn»:
- **Длины волн 850 / 1310 / 1550 нм** как мотив (eyebrow, шкала, цветовые узлы);
- **«Свет в волокне»** — движущиеся световые импульсы по горизонтальным fiber-линиям в тёмных шапках;
- **Циан = фотоника** — точечный второй акцент в данных и активных состояниях;
- **Спектральный разделитель** (`.spectrum-rule`) — тонкая полоса с градиентом длин волн.

### Темы
Светлая и тёмная — **равноправные**, не инверсия друг друга.
- **Тёмная** (`slate-950 #0B0F17`) — родная «приборная» среда.
- **Светлая** (холодный `#F4F6FA`) — чистая datasheet-среда, белые карточки «всплывают» над слегка холодным фоном.
- **Шапки страниц** (`.page-head`) и hero — тема-зависимые: в тёмной теме это глубокий тёмный фотонический блок, в светлой — лёгкая светлая «приборная» поверхность с тёмным текстом и циан-акцентом.
- Переключатель в шапке; выбор хранится в `localStorage['modul-theme']`.

### Сетка и контейнер
- Максимальная ширина контента — **1320px**, горизонтальные поля — 24px (`px-6`).
- Сетка на базе **4px**; основной шаг — 4 / 8 / 12 / 16 / 24 / 32 / 48.

## 3. Токены

Все токены заданы как **HSL-каналы** в CSS-переменных (`assets/tokens.css`) и используются как `hsl(var(--token))`. Формат совместим с shadcn/ui: те же имена переменных, тот же способ подключения через Tailwind-конфиг.

### 3.1 Цвет — светлая тема (`:root`)

| Токен | Значение (HSL) | Назначение |
|---|---|---|
| `--background` | `214 32% 97%` | Фон страницы (холодный, чтобы белые карточки выделялись) |
| `--foreground` | `222 47% 11%` | Основной текст |
| `--card` | `0 0% 100%` | Поверхность карточек |
| `--card-foreground` | `222 47% 11%` | Текст на карточках |
| `--popover` | `0 0% 100%` | Поповеры, меню |
| `--muted` | `216 24% 93%` | Приглушённые поверхности |
| `--muted-foreground` | `220 9% 43%` | Вторичный текст |
| `--subtle` | `215 30% 95%` | Зебра / шапки таблиц (утоплённый фон) |
| `--border` | `218 18% 89%` | Границы |
| `--border-strong` | `220 14% 82%` | Усиленные границы / разделители |
| `--input` | `220 16% 88%` | Границы инпутов |
| `--ring` | `222 89% 46%` | Кольцо фокуса |

**Бренд:**

| Токен | Значение | HEX | Назначение |
|---|---|---|---|
| `--primary` | `222 89% 46%` | `#0A4FD6` | Глубокий оптический синий — действия, фокус |
| `--primary-foreground` | `0 0% 100%` | `#FFFFFF` | Текст на primary |

| `--cyan-muted` | `190 80% 95%` | — | Тинт-поверхности циана |

**Семантика:**

| Токен | Значение | Назначение |
|---|---|---|
| `--success` / `--success-muted` | `142 71% 38%` / `142 60% 95%` | В наличии · протестировано |

**Data-viz ramp** (диаграммы, оптбюджет): `--data-1` … `--data-5` = синий, циан, зелёный, янтарь, фиолетовый.

### 3.2 Цвет — тёмная тема (`.dark`)
Тёмная тема переопределяет те же токены. Ключевые отличия:

| Токен | Значение | HEX |
|---|---|---|
| `--background` | `222 33% 7%` | `#0B0F17` (slate-950) |
| `--card` | `222 28% 9%` | `#10151F` |
| `--primary` | `217 91% 60%` | `#3B82F6` (поднят для тёмного фона) |
| `--cyan` | `188 86% 53%` | ярче для контраста |
| `--border` | `220 20% 18%` | — |

Семантические и data-токены в тёмной теме подняты по яркости (например, `--success: 142 65% 45%`), тени — на чёрной базе с большей альфой.

### 3.3 Типографика
Два семейства:
- **Inter** — интерфейс и текст (нейтральный, плотный). Веса 400 / 500 / 600 / 700.
- **IBM Plex Mono** — артикулы, длины волн, °C, DOM-параметры, числа в таблицах. Веса 400 / 500 / 600.

| Уровень | Размер / насыщенность | Применение |
|---|---|---|
| H1 | 36px / 700 (до 3.4rem в hero) | Заголовок страницы |
| H2 | 30px / 600 | Раздел |
| H3 | 24px / 600 | Подраздел |
| H4 | 18px / 600 | Заголовок карточки |
| H5 / Eyebrow | 14px / 600, uppercase, tracking-wide | Надзаголовок |
| H6 / Label | 11px / 600, uppercase, tracking-wider | Метки |
| Body | 15px, line-height ~1.6 | Абзацы |
| Caption | 13px, muted | Подписи, таблицы |
| Mono | 13px, tabular-nums | Артикулы и параметры |

**Утилиты:**
- `.mono` — IBM Plex Mono + `font-variant-numeric: tabular-nums`;
- `.tnum` — табличные цифры (выравнивание чисел по разрядам);
- Числа в таблицах — всегда `tabular-nums` и по правому краю (`text-right`).
- Дополнительный размер `text-2xs` = `0.6875rem` (11px) с увеличенным трекингом.

### 3.4 Отступы, радиусы, тени

**Отступы** — шкала на базе 4px: `4 · 8 · 12 · 16 · 24 · 32 · 48`.

**Радиусы** (`--radius: 0.5rem` = 8px):
| Класс | Значение |
|---|---|
| `rounded-sm` | 4px (`--radius − 4`) |
| `rounded-md` | 6px (`--radius − 2`) |
| `rounded-lg` | 8px (`--radius`) |
| `rounded-full` | круг / пилюля |

**Тени** (мягкие, на базе `slate-900` в светлой и чёрного в тёмной):
| Токен | Применение |
|---|---|
| `--shadow-xs` | кнопки, мелкие поверхности |
| `--shadow-sm` | карточки в покое |
| `--shadow-md` | карточки при hover, поповеры |
| `--shadow-lg` | модалки, drawer, стеклянные панели |

### 3.5 Оптический язык (CSS, `assets/base.css`)
| Класс | Назначение |
|---|---|
| `.page-head` | Тема-зависимая шапка страницы (тёмная в dark / светлая в light) |
| `.hero` | Тёмная фотоническая поверхность (фон `#070b14`) |
| `.hero-grid` | Инженерная сетка с радиальной маской |
| `.hero-glow` | Размытое сине/циановое свечение |

## 4. Компоненты

Базовые компоненты живут в `design-system.html` (разделы «Компоненты» и «Доменные компоненты»). Ниже — назначение, состояния и структура классов каждого. Код приведён скелетно: реальные элементы используют Tailwind-классы из токенов.

### 4.1 Базовые

**Кнопки (Button)** — высота `h-9` (36px, основная), `h-8` (sm), `h-11` (lg, тач 44px).
- Варианты: `primary` (`bg-primary text-primary-foreground`), `secondary` (`border bg-card`), `muted`, `ghost` (текст-only), `destructive`.
- Состояния: hover (`hover:bg-primary-hover`), disabled (`opacity-40`), иконочная (`h-9 w-9`).

```
<button class="inline-flex h-9 items-center gap-2 rounded-md bg-primary px-4
  text-sm font-medium text-primary-foreground shadow-xs hover:bg-primary-hover">…</button>
```

**Бейджи статуса (Badge)** — наличие и качество.
- Наличие: `success` (в наличии + кол-во), `warning` (под заказ), `destructive` (снято).
- Качество: «протестировано» (галочка), «DOM/DDM», «гарантия», «RoHS».
- Форма: `rounded-full` для статуса, `rounded-md` для меток; точка-индикатор `h-1.5 w-1.5 rounded-full`.

**Чипы фильтров (FilterChip)** — активные (`bg-primary text-primary-foreground` + крестик) и доступные (`border bg-card` + счётчик `mono`). Есть «+ параметр» (dashed) и «сбросить всё».

**Инпут с автодополнением (Combobox)** — `h-10`, иконка поиска, `mono`-значение, kbd-хинт; выпадающий список с группами, активная строка `bg-accent`, артикул + краткие параметры + статус наличия.

**Табы (Tabs)** — нижняя граница, активный таб `border-b-2 border-primary`; счётчики в `mono`.

**Хлебные крошки (Breadcrumbs)** — разделитель `/` цвета `border-strong`, последний сегмент — `mono` артикул.

**Карточка товара (ProductCard)** — изображение модуля (см. 4.3) на градиенте `from-muted to-subtle`, бейдж наличия и «сравнить» поверх; артикул (`mono text-primary`) + «tested»; название; мини-таблица параметров (`mono text-2xs`, key-value); цена «от» + кнопка «+ в спецификацию». Высоты карточек выровнены (`mt-auto` у блока цены).

**Строка тех-таблицы (TableRow)** — зебра (`bg-subtle` у чётных), hover (`bg-accent`), числа `mono tabular-nums text-right`, артикул `text-primary`, шапка `bg-subtle text-xs text-muted-foreground`.

### 4.2 Доменные

**Блок цены (PriceBlock)** — розничная + партнёрская цена, плашка «−68% к OEM <вендор>», тиры объёма (1–9 / 10–49 / 50+) в `mono`-сетке.

**Селект прошивки (FirmwareSelect)** — кодирование под вендора (Cisco / Juniper / Arista / Huawei / MikroTik); инфо-подсказка «программируется на складе, +0 дней»; чипы вендоров.

**Карточка результата ИИ (AIResultCard)** — аватар-ИИ, текст, вложенная карточка модуля с бейджем «подобрано ИИ» (мини-плашка `AI`), цена и «+ в спецификацию».

**Тултип-глоссарий (Tooltip)** — термин с пунктирным подчёркиванием (DOM, FEC, BiDi…); при hover — поповер с пояснением.

**DOM/DDM-таблица (DomTable)** — параметр, текущее значение, пороги low alarm / low warn / high warn / high alarm, статус (`norm` / `warn` / `alarm`) с цветовой точкой.

**Каскад совместимости (CompatCascade)** — три колонки вендор → серия → модель; выбранные элементы `border-primary bg-primary-muted text-primary`.

### 4.3 Иллюстрации модулей (`assets/module-art.js`)
Функция `window.moduleArt(p)` рисует **реалистичный SVG трансивера** по форм-фактору вместо плейсхолдера:
- **SFP / SFP+ / SFP28** — узкий корпус, синяя защёлка (bail), порты LC (циановые «расточки»), зона этикетки с артикулом, золотые контактные «пальцы»;
- **QSFP / QSFP-DD / OSFP** — широкий корпус; порт LR4 (стек LC) или массив точек MPO;
- **DAC/AOC** — корпус справа + медный кабель, выходящий дугой влево;
- **RJ45** — медный порт витой пары.
Цвета берутся из токенов (`--card`, `--cyan`, `--muted`), контакты — фиксированное золото `#C9A24B`. Подключается на каталог, карточку товара и сравнение.

## 5. Экраны

Продукт состоит из **11 экранов** + сквозные AI-инструменты. Все используют единую шапку (логотип, ⌘K-поиск, навигация, ИИ-консультант, переключатель темы, спецификация) и оптический язык.

### 5.1 Главная (`index.html`)
![Главная](img/01-home.png)
**Назначение:** точка входа, экспресс-подбор, демонстрация ассортимента и доверия.
**Состав:**
- утилити-бар (наличие на складе, отгрузка, прайс/API/телефон);
- шапка с поиском по артикулу (триггерит ⌘K) и строкой категорий;
- **hero** (тёмный фотонический / светлый приборный) — заголовок с градиентом, шкала длин волн 850/1310/1550, движущиеся импульсы, статистика, стеклянная карточка экспресс-подбора (вендор → серия → модель → live-превью результата);
- категории по форм-фактору (1G → 400G + WDM/DAC) с иллюстрациями;
- отраслевые подборки (ЦОД, операторы, видеонаблюдение, кампус);
- «популярные решения» (готовые комплекты);
- таблица «популярное на складе» со столбцом «vs OEM»;
- **тёмная полоса доверия** «Инженерное доверие, а не маркетинг» (тест, отгрузка, гарантия, datasheet);
- футер.
**Поведение:** переключение темы, экспресс-подбор ведёт на «Совместимость», плавающая кнопка со-пилота.

### 5.2 Каталог (`catalog.html`)
![Каталог](img/02-catalog.png)
**Назначение:** фасетный поиск по параметрам.
**Состав:** фасетный фильтр (скорость, тип среды, дальность, λ/канал, разъём, °C, DOM) со счётчиками у чекбоксов; сетка/список карточек; сортировка; мобильный offcanvas-фильтр; трей сравнения.
**Поведение:** фильтрация в реальном времени (`assets/catalog.js`), переключение плотности/вида, выбор до 4 модулей в трей сравнения (контент получает нижний отступ под трей), тосты-уведомления.

### 5.3 Карточка товара (`product.html`)
![Карточка товара](img/03-product.png)
**Назначение:** полный datasheet модели.
**Состав:** галерея (реальное изображение модуля, габариты, DOM-консоль как «слайды»), хлебные крошки, блок цены/прошивки, табы (Характеристики · Совместимость · DOM/DDM · Аналоги · Документация), тех-таблица, таблица совместимости, аналоги.
**Поведение:** переключение миниатюр галереи, табы, выбор прошивки, добавление в спецификацию.

### 5.4 Подбор совместимости (`compatibility.html`)
![Подбор совместимости](img/04-compatibility.png)
**Назначение:** подбор модулей под конкретное оборудование.
**Состав:** липкий каскад вендор → тип устройства → модель (слева); справа — совместимые модули **по порт-группам** (Uplink / Downlink) с ролью «Основной / Альтернатива», мин. версией ПО, наличием, ценой; кнопка «+ в комплект».
**Поведение** (`assets/compat.js`): смена вендора пересобирает список моделей; выбор модулей наполняет **липкий трей комплекта** (итог + «Добавить комплект»); «Не нашли модель? Спросить ИИ».

### 5.5 Калькулятор оптбюджета (`calculator.html`)
![Калькулятор оптбюджета](img/05-calculator.png)
**Назначение:** расчёт оптического бюджета линии.
**Состав:** конструктор линии (трансивер, длина волны 850/1310/1550, сегменты кабеля, коннекторы, сварки, аттенюатор, требуемый резерв); справа (live) — вердикт, шкала «потери / бюджет» с порогом резерва, раскладка потерь, экспорт PDF.
**Поведение** (`assets/calculator.js`): пересчёт в реальном времени; вердикт 🟢 работоспособна / 🟠 без запаса / 🔴 не хватает; при дефиците — кнопка «Спросить ИИ».
Модель расчёта: затухание волокна (850 — 3.0; 1310 — 0.35; 1550 — 0.22 dB/км) + коннекторы (0.5 dB/пара) + сварки (0.1 dB) + аттенюатор; запас = бюджет − потери − требуемый резерв.

### 5.6 ИИ-диагностика DOM (`dom-diagnostics.html`)
![ИИ-диагностика DOM](img/06-dom.png)
**Назначение:** разбор вывода `show interface transceiver`.
**Состав:** тёмное консольное поле ввода (слева), результат (справа): вердикт + карточки по портам с 5 метриками (температура, напряжение, ток, Tx, Rx), статусом по порогам и шкалой запаса Rx.
**Поведение** (`assets/dom.js`): regex-парсер (Cisco/Juniper/Huawei/Arista), стриминг ответа, статистика норма/внимание/авария; нераспознанный ввод → подсказка.

### 5.7 Сравнение (`compare.html`)
![Сравнение](img/07-compare.png)
**Назначение:** сравнение до 4 модулей.
**Состав:** колонки модулей (изображение, артикул, цена, −% OEM, CTA) + колонка «Добавить модуль»; строки параметров с подсветкой различий; тумблер «Только различия».
**Поведение** (`assets/compare.js`): добавление через пикер-модалку, удаление колонок, подсветка различающихся строк (жёлтый + точка), фильтр «только различия».

### 5.8 Декодер артикула (`decoder.html`)
![Декодер артикула](img/08-decoder.png)
**Назначение:** разбор названия модуля по сегментам.
**Состав:** поле ввода, цветные чипы-сегменты (форм-фактор / скорость / тип-дальность / направление / канал), список «что значит каждая часть» с пояснениями.
**Поведение** (`assets/decoder.js`): живой разбор по мере ввода; словарь форм-факторов, скоростей, кодов SR/LR/ER/ZR/BX/DR/FR/CWDM/DWDM; примеры в один клик.

### 5.9 Спецификация / корзина (`cart.html`)
![Спецификация](img/09-cart.png)
**Назначение:** сборка заказа и запрос КП.
**Состав:** позиции (кол-во с авто-тиром цены, прошивка, −% OEM, комментарий); сводка (экономия vs OEM, сумма, НДС, итог); промокод; оформление (прикрепить ТЗ, «Запросить КП», «Оформить заказ»); блок BOM-импорта.
**Поведение** (`assets/cart.js`): степперы пересчитывают тир и итог; промокоды `MODUL10` / `PARTNER`; «Запросить КП» → модалка успеха.

### 5.10 Личный кабинет (`account.html`)
![Личный кабинет](img/10-account.png)
**Назначение:** B2B-дашборд партнёра.
**Состав:** боковая навигация; KPI (заказов, оборот, **экономия vs OEM**, активные КП); история заказов со статусами и «повторить»; шаблоны спецификаций; избранное.

### 5.11 Дизайн-система (`design-system.html`)
![Дизайн-система](img/11-design-system.png)
**Назначение:** живой референс системы.
**Состав:** палитра (бренд + семантика + нейтрали), типографика, сетка/отступы/радиусы/тени, базовые и доменные компоненты. Переключатель темы демонстрирует обе темы.

### 6.1 ИИ-со-пилот (drawer)
![ИИ-со-пилот](img/14-copilot.png)

## 8. Перенос в Next.js + shadcn/ui

Интерфейс изначально спроектирован под перенос **без переделки**.

### 8.1 Токены
`assets/tokens.css` уже в формате shadcn (HSL-каналы в `:root` и `.dark`). Перенести содержимое в `app/globals.css` внутри `@layer base { :root { … } .dark { … } }`. Имена переменных совпадают с конвенцией shadcn (`--background`, `--foreground`, `--primary`, `--muted`, `--border`, `--ring` и т.д.), плюс доменные расширения (`--cyan`, `--success`, `--warning`, `--subtle`, `--data-1…5`).

### 8.2 Tailwind-конфиг
`assets/tw-setup.js` — это объект конфигурации (для CDN), который один-в-один переносится в `tailwind.config.ts`:
- `darkMode: 'class'`;
- `theme.extend.colors` маппит токены через `hsl(var(--token))` (включая `primary.hover`, `primary.muted`, `cyan`, `success`, `warning`, `destructive`, `info`, `data.1…5`, `subtle`, `border-strong`);
- `borderRadius` через `var(--radius)`;
- `fontFamily.sans = Inter`, `fontFamily.mono = IBM Plex Mono`;
- `boxShadow` через `var(--shadow-*)`;
- `fontSize.2xs = 0.6875rem`;
- `container` center, max 1320px.

### 8.3 Соответствие компонентов shadcn/ui
| Компонент Modul | Примитив shadcn/ui |
|---|---|
| Button | `button` |
| Badge (статус/качество) | `badge` (+ кастомные варианты) |
| FilterChip | `toggle` / `badge` + кастом |
| Combobox / ⌘K | `command` + `dialog` (cmdk) |
| Tabs | `tabs` |
| Breadcrumbs | `breadcrumb` |
| ProductCard | `card` (композиция) |
| TableRow / DomTable | `table` |
| FirmwareSelect | `select` |
| Tooltip | `tooltip` |
| ИИ-drawer | `sheet` |
| Модалки (КП, пикер) | `dialog` |
| Тосты | `sonner` / `toast` |

### 8.4 Структура переноса
- Шрифты — через `next/font` (Inter, IBM Plex Mono) вместо `<link>`.
- Тема — `next-themes` (класс `dark` на `<html>`), вместо ручного `localStorage`-скрипта.
- Логика страниц (`assets/*.js`) переписывается на React-состояние (`useState`/`useReducer`): фасетный фильтр, каскад, калькулятор, парсер DOM, сравнение, корзина — все они уже изолированы по файлам и оперируют чистыми данными.
- Иллюстрации модулей (`module-art.js`) → React-компонент `<ModuleArt p={…} />`, возвращающий SVG.
- Оптический язык (`base.css`) → перенести в `globals.css` (классы `.page-head`, `.hero`, `.fiber-line`, `.pulse`, `.glass`, `.spectrum-rule` остаются как есть).

### 8.5 Доступность (соблюдено, сохранить при переносе)
- Контраст текста ≥ WCAG AA; видимые кольца фокуса (`:focus-visible`, токен `--ring`).
- Тач-зоны ≥ 44px (lg-кнопки, мобильные элементы).
- Семантические теги, `aria-label` на иконочных кнопках, поддержка `prefers-reduced-motion`.

## 9. Доменные данные

### 9.1 Схема артикула (префикс `MC-`)
```
MC- <форм-фактор><скорость> - <тип/дальность> [-направление|канал]
Пример: MC-SFP10G-LR     = SFP+ 10G 1310нм 10км
        MC-QSFP100G-LR4  = QSFP28 100G CWDM4 10км
        MC-SFP10G-BX-D   = SFP+ 10G BiDi downstream
        MC-QSFP100G-DAC3 = QSFP28 100G DAC пассивный 3м
```

### 9.2 Форм-факторы
| Форм-фактор | Скорость | Каналы | Типичные коды |
|---|---|---|---|
| SFP | 1G | 1 | T, SX, LX, BX, CWDM |
| SFP+ | 10G | 1 | SR, LR, ER, ZR, LRM, BX |
| SFP28 | 25G | 1 | SR, LR (FEC-aware) |
| QSFP+ | 40G | 4 | SR4, LR4, 4×10G breakout |
| QSFP28 | 100G | 4 | SR4, LR4, CWDM4, DAC, AOC |
| QSFP-DD | 400G | 8 | DR4, FR4, SR8 |
| OSFP | 400/800G | 8 | DR4, 2×FR4 |

### 9.3 Коды дальности/среды (декодер)
| Код | Расшифровка | Среда / λ |
|---|---|---|
| SR / SR4 | Short Reach | MMF, 850 нм, ≤ 300–400 м |
| LR / LR4 | Long Reach | SMF, 1310 нм, ≤ 10 км |
| LRM | Long Reach Multimode | MMF, 1310 нм, ≤ 220 м |
| ER / ER4 | Extended Reach | SMF, 1550 нм, ≤ 40 км |
| ZR | Z Reach | SMF, 1550 нм, ≤ 80 км |
| BX | BiDi | одно волокно, разные λ Tx/Rx |
| DR / DR4 | Datacenter Reach | SMF, 1310 нм, ≤ 500 м |
| FR / FR4 | Far Reach | SMF, 1310 нм, ≤ 2 км |
| DAC / AOC | Direct Attach / Active Optical | медь / активный кабель |
| CWDM / DWDM | спектральное уплотнение | сетка 20 нм / ITU 50–100 ГГц |

### 9.4 Расчёт оптбюджета
- Затухание волокна: **850 нм — 3.0**, **1310 нм — 0.35**, **1550 нм — 0.22** dB/км.
- Коннекторная пара — **0.5 dB**; сварной стык — **0.1 dB**; аттенюатор — задаётся.
- `Запас = Бюджет трансивера − Σ потерь − Требуемый резерв`.
- Вердикт: запас ≥ 0 → 🟢; потери ≤ бюджет, но запас < резерва → 🟠; потери > бюджет → 🔴.

### 9.5 DOM/DDM — пороги (диагностика)
| Параметр | Low alarm | Low warn | High warn | High alarm |
|---|---|---|---|---|
| Температура, °C | −5 | 0 | 70 | 75 |
| Напряжение, В | 2.97 | 3.13 | 3.46 | 3.63 |
| Ток смещения, mA | 2 | 3 | 11 | 13 |
| Tx Power, dBm | −8.2 | −5.2 | −1.0 | 0.5 |
| Rx Power, dBm | −14.4 | −11.4 | 0.5 | 2.0 |

### 9.6 Ценообразование
- **Розничная** / **партнёрская** цена + **«экономия vs OEM»** (% и абсолют).
- Тиры объёма: **1–9** (база) / **10–49** (−9%) / **50+** (−17%).
- Промокоды: `MODUL10` (−10%), `PARTNER` (−15%). НДС 20%.
