# Design Brief Prompt — для проработки дизайна Modul comp в Claude

> **Как использовать:** открой новый чат в Claude (claude.ai, желательно с артефактами), вставь текст ниже между линиями `=====`. Claude сгенерит дизайн-систему и HTML-макеты ключевых экранов. Итерируй: «сделай темнее», «покажи мобильную версию каталога», «дай вариант B главной». Когда визуал устроит — приноси сюда (скриншоты + HTML/CSS токены), наденем на рабочее демо (Next.js + Tailwind).

> **Зачем именно так:** дизайн прорабатываем отдельно, потому что визуал — это первое, что увидят ЛПР компании. Сначала «вау» от внешнего вида, потом подкладываем живую логику.

============================ КОПИРОВАТЬ ОТСЮДА ============================

Ты — senior product-дизайнер, специализирующийся на B2B / технических SaaS и e-commerce интерфейсах. Спроектируй дизайн-систему и ключевые экраны для профессионального B2B-интернет-магазина телекоммуникационного оборудования.

## Контекст продукта
Интернет-магазин оптических трансиверов и сетевого оборудования: SFP/SFP+/SFP28/QSFP+/QSFP28/QSFP-DD/OSFP модули, DAC/AOC-кабели, патч-корды, CWDM/DWDM-мультиплексоры, готовые решения. Это **демо-презентация**, цель которой — произвести впечатление профессионализма и глубокого понимания домена.

## Аудитория (для кого дизайн)
Инженеры связи, сетевые администраторы, проектировщики, системные интеграторы, отделы закупок. Это техническая, придирчивая аудитория. Они ценят: точность данных, плотные но читаемые таблицы характеристик, быстрый поиск по параметрам, отсутствие «маркетинговой воды». Дизайн должен внушать **инженерное доверие**, а не выглядеть как потребительский магазин гаджетов.

## Бенчмарк
Главный конкурент-ориентир — **FS.com** (изучи их паттерны B2B-каталога трансиверов и сделай чище/современнее). Для уровня полировки и «продуктовости» ориентируйся на **Linear, Vercel, Stripe Dashboard**. Для подачи технических данных — стилистика хорошего **datasheet / спецификации**: моноширинные акценты для артикулов и параметров, аккуратные таблицы, чёткая типографика.

## Принципы дизайна
1. **Инженерная точность.** Чистая сетка, выверенные отступы, спокойная палитра. Данные — главный герой.
2. **Плотность без хаоса.** Таблицы характеристик и фасетные фильтры могут быть плотными, но обязаны оставаться читаемыми (типографическая иерархия, зебра, выравнивание чисел по правому краю).
3. **Технологичность, не игрушечность.** Сдержанный акцентный цвет (предложи: глубокий синий/циан или электрик — ассоциация с оптикой/фотоникой), нейтральная база, аккуратные тени.
4. **Доверие.** Бейджи «протестировано на оборудовании», статусы наличия, сертификаты — оформить как знаки качества, не как рекламу.
5. **Адаптивность.** Десктоп-first (рабочий инструмент), но мобильная версия обязана быть удобной: фильтры в offcanvas-панели, крупные тач-зоны.

## Что нужно сдать (по порядку, в виде HTML-артефактов на Tailwind CSS)

### 1. Дизайн-система (отдельная страница-гайд)
- Палитра: нейтральные + 1 акцент + семантические (успех/наличие, предупреждение/под заказ, ошибка). Дай HEX и предложи light + dark тему.
- Типографика: шкала (H1–H6, body, caption, mono для артикулов/параметров). Предложи конкретные шрифты (например, Inter / IBM Plex Sans + IBM Plex Mono).
- Сетка и отступы (spacing scale), радиусы, тени.
- Базовые компоненты: кнопки (primary/secondary/ghost), бейджи статуса, чипы фильтров, инпут с автодополнением, карточка товара, строка таблицы характеристик, таб, хлебные крошки.

### 2. Ключевые экраны (полноценные макеты, реалистичные данные про трансиверы — не lorem ipsum)
Приоритет именно такой:
1. **Главная** — хедер (логотип, поиск по артикулу/модели, ЛК, корзина), блок «Экспресс-подбор совместимости» (вендор → серия → модель), карточки категорий (SFP+, QSFP28, DAC, CWDM-мультиплексоры…), отраслевые подборки (ЦОД / операторы / промышленные), популярные решения, футер.
2. **Каталог с фасетным фильтром** — слева фасеты (скорость 1G/10G/25G/100G…, тип среды SR/LR/ER/BiDi/CWDM/DWDM, дальность, длина волны/канал, разъём LC/SC/MPO, температурный диапазон, DOM), сверху сортировка и поиск, сетка/список карточек товара, рядом с каждым чекбоксом — количество товаров. Покажи и мобильный вид с offcanvas-фильтром.
3. **Инструмент «Подбор совместимости»** — последовательный выбор Вендор (логотипы Cisco/Juniper/Huawei/Arista/MikroTik) → тип устройства → модель → таблица совместимых модулей с разбивкой по портам (uplink 10G SFP+, downlink 1G SFP), пометки «Основной модуль / Альтернатива», кнопка «Добавить комплект в корзину».
4. **Карточка товара** — галерея (фото модуля, схема размеров, экран тестера DOM), название вида `SFP+ 10GBASE-LR 1310nm 10km LC DOM`, цена + «запросить оптовую», статус наличия, выбор прошивки (по умолчанию «Универсальный (Generic)»), кнопки В корзину / Купить в 1 клик / Сравнить / Избранное; вкладки: Характеристики (полная тех-таблица), DOM-параметры, Таблица совместимости (вендор/модель/мин. версия ПО), Документы, Аналоги/похожие.
5. **Калькулятор оптического бюджета** — интерактивная форма: добавление сегментов кабеля, коннекторов, сварок, аттенюаторов; выбор трансивера (мощность Tx / чувствительность Rx); визуальный результат — шкала запаса бюджета и вердикт «Работоспособен / Требуется усилитель»; кнопка «Сохранить в PDF».
6. **Сравнение товаров** — до 4 модулей бок-о-бок, группировка строк характеристик, подсветка различий.

## Технические ограничения (важно — дизайн пойдёт в реальную сборку)
- Все макеты — на **Tailwind CSS** (классы прямо в HTML), чтобы их можно было перенести в Next.js + shadcn/ui без переделки.
- Используй CSS-переменные для палитры (design tokens), чтобы light/dark переключались.
- Реалистичные данные домена (настоящие протоколы, длины волн, дальности, артикулы), это критично для доверия аудитории.
- Доступность: контраст, фокус-стейты, размеры тач-зон.

## Формат ответа
Сначала кратко (5–7 строк) опиши концепцию: настроение, палитра, шрифты, почему так — под эту инженерную B2B-аудиторию. Затем выдавай артефакты по одному: сперва дизайн-систему, потом экраны по приоритету. После каждого жди моих правок, прежде чем идти дальше.

Начни с дизайн-системы и концепции главной страницы.
