# tasks/ — Управление задачами проекта Modul comp
# @file: tasks_README

> **Принцип:** ничего не теряется. Все задачи от глобальных до атомарных — в `active.md`. Все выполненные — в `done.md` с описанием решения.

---

## ФАЙЛЫ

| Файл | Назначение |
|---|---|
| `active.md` | **Активные задачи.** Дерево от глобальных (8 этапов) до минимальных. Минимум = одно конкретное действие или один файл |
| `done.md` | **Выполненные задачи.** С датой и описанием что сделали (ссылки на файлы, краткое описание решения) |
| `README.md` | Этот файл — методология ведения |

---

## СТРУКТУРА `active.md` — ИЕРАРХИЯ

```
ЭТАП (глобальный, из 8 слоёв)
└── Подсистема
    └── Задача
        └── Подзадача
            └── Атомарное действие
```

Глубина не ограничена — детализируем пока есть смысл.
**Минимальная задача:** одно конкретное действие или один создаваемый файл.

## СТАТУСЫ ЗАДАЧ (маркдаун-чекбоксы)

| Маркер | Значение |
|---|---|
| `- [ ]` | Не начата |
| `- [/]` | В работе |
| `- [x]` | Выполнена → перенос в `done.md` |
| `- [?]` | Заблокирована — нужно решение Евгения |
| `- [-]` | Отменена (с пометкой почему) |

---

## ПРАВИЛА РАБОТЫ

### 1. Любая работа = задача из `active.md`

Перед началом работы:
1. Проверь `active.md` → найди соответствующую задачу
2. Если задачи нет — заведи (с подтверждением Евгения, если задача глобальная или меняет план)
3. Поставь статус `[/]` (в работе)

### 2. Атомарность

Если задача делается дольше 30 минут — разбей на подзадачи.
Без атомарности теряется контроль и нечего фиксировать в done.

### 3. Уточнения у Евгения

Если не хватает данных:
1. Статус задачи → `[?]`
2. В комментарии — конкретный вопрос
3. После ответа Евгения — продолжаем

### 4. Перенос в `done.md`

После выполнения:
1. Добавь запись в `done.md` с датой и описанием что сделали
2. Помети задачу `[x]` в `active.md`
3. При следующем апдейте `active.md` — удали `[x]` задачи (они уже в done.md)

### 5. Связь с SESSION_CONTEXT

| Файл | Что фиксирует |
|---|---|
| `SESSION_CONTEXT.md` → `@last` | Краткий итог сессии |
| `tasks/done.md` | Полная история ВСЕХ задач с разбивкой по дням |
| `tasks/active.md` | Что предстоит |

Файлы дополняют друг друга, не дублируют.

## ПЕРЕИМЕНОВАНИЕ ЗАДАЧ

Если задача переформулируется — старая идёт в done.md как отменённая (`[-]`) с пояснением, новая — в active.md.

Не правим формулировку «по тихому» — теряется история.

---

## ГЛАВНЫЙ ПРИНЦИП

> **Полный анализ + полное понимание.** Каждая задача расписана подробно, с учётом связей с другими задачами. Если задача связана с другой — указываем (`см. 1.2.3`).
