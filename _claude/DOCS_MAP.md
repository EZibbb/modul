# DOCS_MAP — Какой документ для чего
# @file: DOCS_MAP

> **Решение D006:** при любом сомнении куда что записывать — открой этот файл. Один документ ≠ свалка. Каждый имеет цель.

---

## Принцип

> **Одна цель → один документ.** Если запись подходит к 2+ документам — значит цель размыта. Уточни.

---

## Карта документов

### Системные (мозг проекта)

| Документ | Цель | Что туда писать | Что НЕ писать |
|---|---|---|---|
| `CLAUDE.md` | Главный маршрутизатор, читается первым | Ссылки на ключевые файлы + правила выжимка | Подробности процесса |

## Типовые ситуации

| Хочу записать | Куда |
|---|---|
| Новое архитектурное решение | `_claude/research/decisions.md` → новый D### |
| Кандидат на решение (надо обсудить) | `_claude/research/decisions.md` → D-кандидаты |
| Ссылку на полезную статью / видео | `_claude/research/inbox.md` |
| Идею к разбору позже | `_claude/research/inbox.md` |
| Новую активную задачу | `_claude/tasks/active.md` |
| Завершённую задачу | `_claude/tasks/done.md` + `[x]` в active.md |
| «Что мы сегодня делали» | `_claude/SESSION_CONTEXT.md` → LOG + обновить `@last` |
| Описание новой фичи продукта | `_claude/concept/03_features/...` (по домену) |
| Новое правило работы со мной (Claude) | Memory (`save user memory`) или `INSTRUCTIONS.md` если общее |
| Описание нового скилла | `_claude/SKILL_MAP.md` |
| Заметку «не забыть на потом» | `_claude/research/inbox.md` |
