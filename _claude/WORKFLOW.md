# WORKFLOW — Шаблон выполнения задачи
# @file: WORKFLOW

> Используется при каждой нетривиальной задаче.
> Связано с правилами в `_claude/INSTRUCTIONS.md` (ГЛАВНОЕ ПРАВИЛО, 6 шагов отчёта, 3 фазы верификации).

---

## УРОВНИ ЗАДАЧ

| Уровень | Признак | Что делаем |
|---|---|---|
| Мелкая | 1–2 файла, очевидно (опечатка, мелкая правка) | Без чеклиста. Только обновить `@last`. |
| Средняя | 3–5 файлов, есть варианты | Сокращённый чеклист (см. ниже) + 6-шаговый отчёт. |
| Крупная | Новый блок концепции / архитектура / новый модуль | Полный чеклист + 6-шаговый отчёт + бэкап (если применимо по правилу из INSTRUCTIONS.md). |
| Критическая | Необратимо / удаление / выход за Modul comp | СТОП → подтверждение Евгения. |

---

## ПОЛНЫЙ ЧЕКЛИСТ (крупная задача)

```
## ЗАДАЧА: [название]
## ДАТА: [YYYY-MM-DD]
## РЕЖИМ: [ОБСУЖДЕНИЕ / ПЛАНИРОВАНИЕ / ПРОЕКТИРОВАНИЕ / РАЗРАБОТКА / РЕВЬЮ / АУДИТ]
## УРОВЕНЬ: крупная

### АНАЛИЗ
- [ ] Прочитал задачу
- [ ] Связанные файлы: [список]
- [ ] Сложность: [низкая / средняя / высокая]
- [ ] Риски: [что может пойти не так]

### ВАРИАНТЫ РЕАЛИЗАЦИИ (если применимо)
1. [Вариант А] — плюс: [...]  минус: [...]
2. [Вариант Б] — плюс: [...]  минус: [...]
→ Выбран: [...]   Причина: [...]

### ШАГИ ВЫПОЛНЕНИЯ
- [ ] Шаг 1
- [ ] Шаг 2
- [ ] ...
- [ ] Последний шаг: верификация (3 фазы + дыры)

### ВЕРИФИКАЦИЯ (3 фазы + дыры)
- [ ] Фаза 1 — внутренняя (то что сделал)
- [ ] Фаза 2 — перекрёстная (соседние файлы и связи)
- [ ] Фаза 3 — полнота (всё из обсуждения отражено)
- [ ] Проверка на дыры (слабое место, что сломается в реальности)

### ФИКСАЦИЯ
- [ ] Обновил `@last` в SESSION_CONTEXT.md (старый @last → LOG)
- [ ] Перенёс задачу в `tasks/done.md` (если завершён блок)
- [ ] Обновил NAV_INDEX.md (если создал/удалил файлы)
- [ ] Бэкап (если применимо по правилу из INSTRUCTIONS.md)

### ИТОГ
Сделано: [что именно]
Файлы: затронуто N: [список]
Следующий шаг: [что дальше]
```

## ОБЯЗАТЕЛЬНЫЙ 6-ШАГОВЫЙ ОТЧЁТ ПОСЛЕ БЛОКА

После выполнения любого блока — публикую Евгению по форме (детали в `INSTRUCTIONS.md`):

```
✅ Шаг 1 — Анализ        (что перечитал, с чем сравнил, что нашёл)
✅ Шаг 2 — Ошибки        (таблица или «не найдено»)
✅ Шаг 3 — Исправления   (что исправил, или «не требуется»)
✅ Шаг 4 — Повторная проверка  (результат после правок)
✅ Шаг 5 — Полнота       (всё ли из обсуждения отражено)
✅ Шаг 6 — Финал         (затронуто файлов: N)
```

**Запрещено** объединять шаги, пропускать шаги, писать «Готово» без всех 6.

## КАК ОБНОВИТЬ @last ПОСЛЕ ЗАДАЧИ

В `_claude/SESSION_CONTEXT.md`:
1. Старый `## @last` → перенести в `## LOG` с датой и номером сессии
2. Новый `@last` сверху, формат:

```
## @last
Дата: YYYY-MM-DD
Сессия: N
Задача: [что делали]
Контекст: [почему — мотивация]
Результат: [что создали / изменили / решили]
Файлы: [затронуто]
Следующий шаг: [что дальше]
```

## ПРАВИЛО ВЕРИФИКАЦИИ (короткая форма)

После каждой задачи — 4 вопроса:
1. Задача выполнена полностью? (не частично)
2. Данные / логика правдивы? (не выдумал)
3. Соседние части не сломал?
4. Зафиксировал в `@last`?

Если хоть один ответ «нет» — НЕ отмечаю как завершённое.
