# I. Главная страница — спецификация (11 полей)

> Композиция из блоков других модулей. Первое впечатление ЛПР — должна сразу показать killer-фичи.

## 1. ID + название
**I. Главная** — лендинг-композиция: хедер, экспресс-подбор, категории, подборки, отраслевые входы, преимущества, статьи, футер.

## 2. Краткое описание
Витрина первого экрана: сразу даёт поиск, экспресс-подбор совместимости и быстрые входы в каталог по разным осям.

## 3. Цель
За 10 секунд показать инженеру/ЛПР: «здесь технически, удобно, есть подбор совместимости и ИИ». Маршрутизировать в каталог/инструменты.

## 4. Triggers / точки входа
- Корень `/`, логотип, переходы извне.

## 5. Описание процесса
Блоки сверху вниз (ТЗ 1.1):
1. **Хедер** — логотип, поиск по артикулу/модели (с автодополнением, A), ЛК, корзина, кнопка ИИ-консультанта (L).
2. **Экспресс-подбор совместимости** (виджет B1) — вендор→серия→модель.
3. **Карточки ключевых категорий** — SFP+, QSFP28, DAC, CWDM-муксы, патч-корды, промышленные.
4. **Популярные решения** — сценарии (ЦОД 10/25/100G, 5G CPRI, DWDM-магистраль, PON).
5. **Преимущества** — тестирование на оборудовании, программирование под вендора, техподдержка.
6. **Отраслевые подборки** — ЦОД / операторы / промышленные (лендинг-фильтры, A).
7. **Последние статьи** (G4).
8. **Футер** — разделы, контакты, сервис/документация, политика.

## 6. UI/UX
- Чистый первый экран: поиск + экспресс-подбор крупно.
- Карточки категорий с иконками форм-факторов.
- Адаптив: блоки в одну колонку на мобайле.

## 7. Данные (модели / сущности / API)
- Переиспользует API: suggest (A), compat-виджет (B), категории (K), статьи (G).
- Контент блоков (преимущества, решения) — статический конфиг/seed.

## 8. Доступность
Публично.

## 9. Лимиты
Демо: 2–3 статьи, фикс-набор сценариев/отраслей.

## 10. Этап реализации (тир)
- **P0:** хедер + поиск + экспресс-подбор + карточки категорий.
- **P1:** популярные решения · отраслевые подборки · преимущества · статьи.

## 11. Приоритет
**P0** (верхняя часть — критична для первого впечатления).
