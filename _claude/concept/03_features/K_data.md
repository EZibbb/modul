# K. Данные / seed — спецификация (11 полей)

> Фундамент всего: на богатой модели товара держатся фасетный поиск (A), карточка (C), ИИ-слой (L). Делаем первым в коде.

## 1. ID + название
**K. Данные / seed** — модель данных товара, схема атрибутов под фасеты, матрица совместимости, генерация seed-каталога.

## 2. Краткое описание
Гибкая модель «товар + атрибуты», управляющая фасетным поиском и подачей характеристик, плюс реалистичный seed-каталог трансиверов и матрица совместимости — чтобы продукт работал «как настоящий» без 1С.

## 3. Цель
Дать остальным модулям реальные, доменно-достоверные данные. Без качественного seed ИИ-подбор и фасеты выглядят пусто/фейково — а доверие инженерной аудитории держится именно на точности данных.

## 4. Triggers / точки входа
- Seed-скрипт при инициализации БД (`prisma db seed`).
- CSV-импорт товаров и матрицы совместимости (имитация выгрузки из 1С/поставщиков).
- Источник данных для всех API: каталог, карточка, совместимость, ИИ-RAG.

## 5. Описание процесса
**Модель товара — гибрид:** общие колонки (для всех) + JSONB-атрибуты (специфичные для категории) + словарь определений атрибутов, который управляет фасетами динамически. Категория задаёт, какие атрибуты показывать и фильтровать.

**Атрибуты-определения** делают фасеты не захардкоженными: добавил атрибут в словарь с `facetable=true` → он сам появляется в фильтре нужных категорий с правильным UI (чекбокс/диапазон/селект).

**Импорт:** CSV → upsert по `sku`. Товары и связи совместимости — отдельными файлами. Резолв Vendor/Series/DeviceModel по имени.

## 6. UI/UX
Не имеет своего экрана (фундамент). Но определяет: состав фасетов в каталоге, структуру тех-таблицы в карточке, поля DOM-блока. Качество seed = насколько «живым» выглядит весь сайт.

## 7. Данные (модели / сущности / API)
**Category**(id, parentId, name, slug, formFactor?, attributeKeys[]) — дерево из ТЗ 2.1.
**Brand**(id, name) — производитель/чип модуля (≠ Vendor совместимости).
**AttributeDefinition**(key, label, unit, type[enum/range/bool], filterUi[checkbox/range/select], facetable, categoryKeys[]) — словарь, управляет фасетами.
**Product**(
  id, sku, mpn, name(генерится), categoryId, brandId,
  formFactor, speedGbps, mediaType[SR/LR/ER/ZR/BiDi/CWDM/DWDM/copper],
  reachM, wavelengthNm | dwdmChannel, connector[LC/SC/MPO/RJ45/E2000],
  tempRange[com/ext/ind], domSupport[bool],
  txPowerMin/Max, rxSensitivity, laserType[DFB/FP/VCSEL], fiberType,
  powerConsumptionW, dimensions, weightG, certifications[],
  priceBase, pricePartner?, stockStatus[in/order], leadTimeDays,
  images[], documents[],
  firmwareOptions(JSON: ["Generic", "Cisco", "Juniper"…], default "Generic"),
  attributes(JSONB — спец. поля категории)
)
**+ совместимость** из модуля B (Vendor/DeviceSeries/DeviceModel/PortGroup/Compatibility).

**CSV-форматы:**
- товары: `sku;mpn;category;brand;name;speed;mediaType;reach;wavelength;connector;temp;dom;price;stock;…`
- совместимость: `sku;vendor;series;deviceModel;deviceType;portGroup;role;minSwVersion;note;tested`

**Прошивки (D-007):** Вариант А — один SKU, список кодировок в `firmwareOptions`, цена не зависит от прошивки. Заложить расширение до Варианта Б (отдельные SKU).

## 8. Доступность
Фундамент — данные публичны; `pricePartner` (оптовая) видна только авторизованным B2B.

## 9. Лимиты (seed для демо)
- Категории: полное дерево ТЗ 2.1 (трансиверы/DAC/патч-корды/CWDM-DWDM/решения).
- Товары: **~80–120** реалистичных позиций (SFP/SFP+/SFP28/QSFP+/QSFP28 + DAC/AOC + патч-корды + 2–3 CWDM/DWDM-мукса).
- Бренды совместимости: ~6 (Cisco, Juniper, Huawei, Arista, MikroTik, HP/Aruba, Dell).
- DeviceModel: ~6 вендоров × 3–5 серий × 3–5 моделей, с порт-группами.
- Связи Compatibility: ~50–100 модулей, по несколько моделей на каждый.
- База знаний seed: ITU CWDM (G.694.2) и DWDM (G.694.1) сетки, глоссарий — для справочника (G) и RAG (L).

## 10. Этап реализации (тир)
- **P0:** модель Product + Category + AttributeDefinition · seed ~80–120 товаров · совместимость (с B) · ITU-сетки.
- **P1:** CSV-импортер (демонстрация «как зальётся из 1С») · documents/изображения · pricePartner.

## 11. Приоритет
**P0, наивысший** — делается раньше всех остальных модулей (фундамент). Без него A/C/L пустые.

## Аудит на дыры
| Категория | Находка | Действие |
|---|---|---|
| Критическое | Разные атрибуты у SFP vs DAC vs патч-корд | гибрид: core-колонки + JSONB + AttributeDefinition управляет фасетами |
| Критическое | Фейковые данные убивают доверие | seed строго доменно-достоверный (реальные λ, дальности, протоколы, MPN-паттерны) |
| Среднее | Brand (производитель) ≠ Vendor (совместимость) | разные сущности, не путать |
| Среднее | DWDM канал vs длина волны как фасет | хранить и channel, и nm; фасет по обоим |
| Слабое (W) | Генератор seed должен быть детерминированным (стабильная демо) | W03 → фиксированный seed/слепок |
| Слабое (W) | Картинки модулей (где взять для демо) | W04 → плейсхолдеры/единый стиль карточек |
| Незакрытое | Вариант прошивок А/Б | D-007 — заложен Вариант А с расширением |
| Незакрытое | Маппинг полей при реальном импорте из 1С (CommerceML) | roadmap-заметка, не для seed |
