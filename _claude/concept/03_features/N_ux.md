# N. UX-слой — спецификация (11 полей)

> Ощущение «продукта» уровня Linear/Vercel, а не «магазина». Дешёвый лоск с большим эффектом на впечатление ЛПР.

## 1. ID + название
**N. UX-слой** — N1 Cmd+K быстрый поиск · N2 тёмная тема · N3 сохранение/шаринг конфигураций.

## 2. Краткое описание
Сквозные UX-механики, которые делают сайт быстрым и «продуктовым»: командная палитра, переключение темы, сохранение и шаринг подборок/конфигураций ссылкой.

## 3. Цель
Создать у технической аудитории и ЛПР ощущение зрелого современного продукта. Инженеры ценят клавиатурную навигацию и тёмную тему — это сигнал «сделано для нас».

## 4. Triggers / точки входа
- N1: `Cmd/Ctrl+K` на любой странице.
- N2: переключатель в хедере (+ авто по системной теме).
- N3: кнопка «Сохранить/Поделиться» в сравнении (D), подборе совместимости (B2), конфигураторе линии (M4), ИИ-подборе (L2).

## 5. Описание процесса
**N1 Cmd+K:** командная палитра — мгновенный поиск по артикулу/товару/категории + быстрые действия («открыть калькулятор», «подбор совместимости», «перейти в корзину»). Использует `search/suggest` (A).
**N2 Тёмная тема:** токены оформления через CSS-переменные (заложено в дизайне, D-005). Сохранение выбора в localStorage, авто по `prefers-color-scheme`.
**N3 Сохранение/шаринг:** любая «подборка» (сравнение, комплект совместимости, конфигурация линии) сериализуется в URL/короткую ссылку → коллега открывает то же состояние. Для авторизованных — сохранение в ЛК (мост к F).

## 6. UI/UX
- N1 — модалка-палитра (shadcn Command), группы результатов, клавиатурная навигация, недавнее.
- N2 — плавный переход, без «вспышки» при загрузке (SSR-aware).
- N3 — диалог «Скопировать ссылку» + (для авторизованных) «Сохранить в кабинет».

## 7. Данные (модели / сущности / API)
- N1: переиспользует `GET /api/search/suggest` (A) + статический реестр команд.
- N2: без БД (CSS-переменные + localStorage).
- N3: состояние в query-params/короткий код; `SavedConfig`(id, userId?, type[compare/compat/line], payload, createdAt) для сохранённых в ЛК.

## 8. Доступность
Публично (N1/N2 для всех; N3-шаринг ссылкой — всем, сохранение в кабинет — авторизованным).

## 9. Лимиты
- N1 — результаты ограничены (топ N + «показать все в каталоге»).
- N3 — размер сериализованного состояния (длинные комплекты → короткий код в БД).

## 10. Этап реализации (тир)
- **P0:** N1 Cmd+K · N2 тёмная тема (закладываются с дизайн-системы).
- **P1:** N3 шаринг ссылкой.
- **P2:** N3 сохранение в ЛК (после F).

## 11. Приоритет
**P0** для N1/N2 (дешёвый лоск, ставим сразу с UI-каркасом), **P1/P2** для N3.

## Аудит на дыры
| Категория | Находка | Действие |
|---|---|---|
| Среднее | Тёмная тема ломает datasheet-таблицы/DOM-скрины | проверить контраст обеих тем на тех-таблицах; токены из дизайн-системы (D-005) |
| Среднее | N3 шаринг пересекается с URL-состоянием фасетов (W06) | единый подход: состояние в query-params везде |
| Слабое (W) | FOUC при тёмной теме на SSR | W11 → inline-скрипт темы до гидрации |
| Незакрытое | Короткие ссылки требуют хранилища | при N3-P2: таблица SavedConfig (после F) |
