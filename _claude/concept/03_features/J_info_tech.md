# J. Инфо и технические страницы — спецификация (11 полей)

> Контентные и системные страницы + сквозное SEO. В основном P2, но SEO JSON-LD закладываем раньше.

## 1. ID + название
**J. Инфо / тех** — J1 инфо-страницы (О компании, Доставка/оплата, Контакты) · J2 системные (404, поиск по сайту, карта сайта) · J3 SEO-микроразметка (JSON-LD).

## 2. Краткое описание
Информационные разделы, системные страницы и сквозная структурированная разметка для поисковиков.

## 3. Цель
Завершённость продукта (доверие) + техническое SEO как сигнал зрелости.

## 4. Triggers / точки входа
- Футер, меню, прямые ссылки, поисковые системы.

## 5. Описание процесса
**J1:** О компании (история, компетенции, сертификаты, лаборатория тестирования), Доставка/оплата (условия РФ/СНГ, ТК, реквизиты), Контакты (адрес офиса/склада, форма обратной связи, телефон, email).
**J2:** 404, страница результатов поиска по сайту (мост к A), HTML/XML карта сайта.
**J3 SEO JSON-LD (schema.org):** Product+Offer+brand+additionalProperty(Compatible Vendors) на карточках (C), BreadcrumbList, Article/FAQ (G), SoftwareApplication (калькулятор/инструменты G/M).

## 6. UI/UX
- Контент-страницы — типографика-ориентированные.
- Контакты — карта + форма.
- 404 — полезная (поиск + быстрые ссылки).

## 7. Данные (модели / сущности / API)
- **Page**(slug, title, body) — статический контент/seed или MDX.
- **ContactRequest**(contact, message) — `POST /api/contact`.
- JSON-LD генерируется из данных товара/статей (не отдельная сущность).

## 8. Доступность
Публично.

## 9. Лимиты
Демо: базовый контент-заполнитель, реальные реквизиты компании — позже.

## 10. Этап реализации (тир)
- **P1:** J3 SEO JSON-LD (закладывается с карточкой/каталогом — дёшево, сигнал зрелости).
- **P2:** J1 инфо-страницы · J2 системные (кроме 404 — P1).

## 11. Приоритет
**P1** (J3 SEO, 404), **P2** (контент-страницы).

| Категория | Находка | Действие |
