# H. Сервис и поддержка — спецификация (11 полей)

> Услуги, отличающие компанию: программирование модулей, тестирование, кастомизация. Контент + формы заявок.

## 1. ID + название
**H. Сервис и поддержка** — H1 программирование (кодировка) модулей · H2 тестирование на оборудовании заказчика · H3 кастомизация.

## 2. Краткое описание
Раздел услуг с описанием процесса и формами заявок: прошивка под вендора, проверка совместимости на железе заказчика, нанесение логотипа/серийников.

## 3. Цель
Показать, что компания не просто продаёт коробки, а оказывает инженерные услуги — сильный B2B-аргумент и связка с прошивкой (D-007).

## 4. Triggers / точки входа
- Меню «Сервис и поддержка».
- Из карточки товара (выбор прошивки → «подробнее об услуге», H1).
- Отраслевые лендинги.

## 5. Описание процесса
**H1 Программирование:** описание процесса кодировки, список поддерживаемых вендоров, форма заявки.
**H2 Тестирование:** возможность отправить свой коммутатор/маршрутизатор на проверку совместимости — описание + форма.
**H3 Кастомизация:** нанесение логотипа, серийных номеров под проект — описание + форма.

## 6. UI/UX
- Страницы услуг с понятным описанием шагов (1-2-3) и CTA-формой.
- Связь с карточкой: пометка «программируется на складе» → ссылка сюда.

## 7. Данные (модели / сущности / API)
- **ServiceRequest**(id, type[coding/testing/custom], contact, payload, status).
- `POST /api/service-request`.
- Список поддерживаемых вендоров — из K (Vendor).

## 8. Доступность
Публично (формы — для всех; статус заявки — в ЛК, P2).

## 9. Лимиты
Демо: формы создают заявку (без реального воркфлоу обработки).

## 10. Этап реализации (тир)
- **P2:** все H (контент-страницы + формы). Для питча достаточно показать наличие.

## 11. Приоритет
**P2** (важны как контент-доказательство экспертизы, не интерактив).
