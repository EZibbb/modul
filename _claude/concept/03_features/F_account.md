# F. Личный кабинет (B2B) — спецификация (11 полей)

> Иерархические корп-аккаунты и индивидуальные цены — частично roadmap (зависят от реальной CRM/прайсинга компании).

## 1. ID + название
**F. Личный кабинет** — F1 профиль/реквизиты · F2 иерархические аккаунты · F3 история заказов · F4 избранное и шаблоны · F5 уведомления · F6 оптовые/индивид. цены.

## 2. Краткое описание
Кабинет B2B-клиента: реквизиты, история закупок с повтором, избранное/шаблоны списков, управление сотрудниками, индивидуальные цены.

## 3. Цель
Поддержать регулярные B2B-закупки (повтор заказа, шаблоны, командные аккаунты) и показать зрелость B2B-модели.

## 4. Triggers / точки входа
- «Личный кабинет» в хедере (после авторизации).
- Сохранение из корзины/сравнения/совместимости/ИИ-подбора в избранное/шаблоны.

## 5. Описание процесса
**F1 Профиль:** компания, реквизиты, контакты.
**F2 Иерархические аккаунты:** админ компании управляет доступом сотрудников, видит их заказы.
**F3 История заказов:** таблица с фильтром по дате/статусу, повтор заказа в 1 клик, скачивание счёта/УПД.
**F4 Избранное и шаблоны:** сохранённые товары, сохранённые списки совместимости/конфигурации (мост к N3), шаблоны типовых закупок.
**F5 Уведомления:** о поступлении товара, акциях.
**F6 Цены:** индивидуальные/оптовые цены для авторизованных (мост к A/C/E).

## 6. UI/UX
- Дашборд с разделами (профиль/заказы/избранное/команда/уведомления).
- Таблицы с фильтрами, кнопки повтора/скачивания.

## 7. Данные (модели / сущности / API)
- **User**(id, email, role[admin/member], companyId).
- **Company**(id, name, inn, requisites, priceTier?).
- **Favorite**, **SavedList/Template**(items), **SavedConfig** (общая с N3).
- **Order** (из E) — связь с user/company.
- Auth: NextAuth (email/пароль для демо).

## 8. Доступность
Только авторизованным. F2/F6 — для админов компании / по тарифу цен.

## 9. Лимиты
Демо: 1–2 предзаведённых компании-аккаунта с историей для показа.

## 10. Этап реализации (тир)
- **P1:** F1 профиль · F3 история (с seed-заказами) · F4 избранное/шаблоны.
- **P2:** F2 иерархические аккаунты · F5 уведомления · F6 индивид. цены (зависят от реальной CRM — roadmap).

## 11. Приоритет
**P1** (F1, F3, F4 — для демонстрации флоу), **P2** остальное.
