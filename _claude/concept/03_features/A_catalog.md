# A. Каталог и поиск — спецификация (11 полей)

> Основная витрина. Держится на модели данных K (AttributeDefinition → фасеты) и пересекается с B (фильтр по совместимости).

## 1. ID + название
**A. Каталог и поиск** — дерево категорий, фасетный фильтр, поиск с автодополнением, сортировка.

## 2. Краткое описание
Параметрический каталог трансиверов: слева динамические фасеты, сверху поиск и сортировка, в центре карточки товаров. Фильтры обновляют выдачу без перезагрузки и показывают счётчики.

## 3. Цель
Дать инженеру найти нужный модуль по техническим параметрам за секунды («10G, 1310нм, 10км, LC, DOM»), а не листать сотни позиций. Это базовый рабочий сценарий магазина.

## 4. Triggers / точки входа
- Меню «Каталог» / карточки категорий на главной.
- Переход из виджета совместимости: `/catalog?compatibleWith=<deviceModelId>`.
- Поиск из хедера (по артикулу/модели/характеристикам).
- Отраслевые лендинги (ЦОД/операторы/PON) → каталог с пресет-фильтром.

## 5. Описание процесса
**Дерево категорий** (ТЗ 2.1): Трансиверы → SFP/SFP+/SFP28/QSFP+/QSFP28/… → подтипы (медь/MM/SM/BiDi/CWDM/DWDM/промышл.). Категория задаёт активный набор фасетов через `Category.attributeKeys` + `AttributeDefinition`.

**Фасетный фильтр (динамический):** состав зависит от категории. Типовой набор: скорость, тип среды/стандарт, дальность, длина волны/канал, разъём, температура, DOM, совместимость с вендором. Каждый фасет рендерится по `filterUi` (checkbox/range/select). Рядом со значением — **счётчик товаров** (агрегация). Изменение фасета → AJAX-обновление выдачи + URL (шарабельно).

**Поиск:** по артикулу, названию, модели совместимости + характеристики в свободной форме («10км 1310нм SFP+»). Автодополнение (артикул/категория/модель). Для P0 — поиск по БД (Prisma + нормализация); полнотекст/Meilisearch — P2 (на объёме seed не нужен).

**Сортировка:** цена, популярность, скорость, дальность.

## 6. UI/UX
- Десктоп: фасеты слева (sticky), сетка/список карточек, переключатель вида.
- Мобайл: фасеты в offcanvas-панели (кнопка «Фильтры N»), крупные тач-зоны.
- Активные фильтры — чипами сверху, «сбросить всё».
- Карточка товара в списке: фото, название, ключевые 3–4 параметра, цена, наличие, кнопки «В сравнение/В корзину».
- Пустая выдача — не тупик: «ослабить фильтр» + CTA к ИИ-консультанту.

## 7. Данные (модели / сущности / API)
Опирается на K: `Product`, `Category`, `AttributeDefinition`, `Compatibility`.
API:
- `GET /api/catalog?category=&speed=&media=&reach=&wavelength=&connector=&temp=&dom=&compatibleWith=&sort=&page=` → { items, facets:[{key,values:[{value,count}]}], total }
- `GET /api/search/suggest?q=` → автодополнение.
Фасетные счётчики — агрегирующий запрос (Prisma groupBy) с учётом уже применённых фильтров.

## 8. Доступность
Публично. Авторизованным B2B — `pricePartner` вместо базовой цены в карточках.

## 9. Лимиты
- Пагинация (например, 24/стр).
- На seed ~80–120 товаров агрегации дёшевы; индексы по фильтруемым колонкам.

## 10. Этап реализации (тир)
- **P0:** дерево категорий · фасеты из AttributeDefinition · счётчики · сортировка · поиск по БД + автодополнение · фильтр `compatibleWith`.
- **P1:** отраслевые лендинги-пресеты · переключатель сетка/список · сохранение фильтра.
- **P2:** Meilisearch/полнотекст (если каталог вырастет).

## 11. Приоритет
**P0** — после K и B (нужны данные и связь совместимости).

## Аудит на дыры
| Категория | Находка | Действие |
|---|---|---|
| Критическое | Фасетные счётчики при комбинации фильтров | агрегация с учётом активных фильтров (groupBy), кэш на категорию |
| Среднее | Состав фасетов разный по категориям | управляется `Category.attributeKeys` + AttributeDefinition (из K) |
| Среднее | Диапазонные фасеты (дальность, λ) vs чекбоксы | тип фасета в AttributeDefinition.filterUi (range/checkbox) |
| Среднее | Совместимость как фасет внутри каталога | поле с автодополнением → `compatibleWith` (мост к B) |
| Слабое (W) | Синонимы/опечатки в поиске («1310 nm» vs «1310нм») | W05 → нормализация запроса |
| Слабое (W) | URL-состояние фильтров для шаринга/SEO | W06 → фильтры в query-params (заложить сразу) |
| Незакрытое | Полнотекстовый движок (Meilisearch) — когда | при росте каталога, P2 |
