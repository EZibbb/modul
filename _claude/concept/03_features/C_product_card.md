# C. Карточка товара — спецификация (11 полей)

> Глубина проработки = доверие инженера. Опирается на K (атрибуты), B (совместимость), пересекается с D (сравнение), E (корзина), L (ИИ).

## 1. ID + название
**C. Карточка товара** — страница модуля: галерея, тех-таблица, DOM, совместимость, прошивка, документы, аналоги.

## 2. Краткое описание
Полная техническая карточка трансивера в стиле datasheet: все характеристики, диагностические DOM-параметры, таблица протестированной совместимости, выбор кодировки и блок аналогов.

## 3. Цель
Дать инженеру принять решение прямо здесь: подходит ли модуль, встанет ли в его оборудование, какие альтернативы. Снять необходимость искать PDF-спеку на стороне.

## 4. Triggers / точки входа
- Клик по товару в каталоге/поиске/сравнении/аналогах.
- Прямая ссылка `/product/<sku>` (шаринг, SEO, ИИ-рекомендация).
- Переход из таблицы совместимости (B2).

## 5. Описание процесса
Блоки: хлебные крошки + артикул → галерея → основная инфо (название, цена, наличие, прошивка, кнопки) → вкладки: **Характеристики**, **DOM-параметры**, **Совместимость**, **Документы**, **Аналоги**.

**Название генерится** из атрибутов: `SFP+ 10GBASE-LR 1310nm 10km LC DOM`.
**Выбор прошивки** (D-007, Вариант А): селект `firmwareOptions`, по умолчанию «Универсальный (Generic)», пометка «Программируется на складе». Цена не меняется.
**Таблица совместимости** (из B3): протестированные вендоры/модели с поиском внутри, мин. версия ПО, примечания, бейдж `tested`. Ссылка «Проверить совместимость с вашим оборудованием» → виджет (B1), предзаполненный.

## 6. UI/UX
- Слева галерея (фото модуля, схема размеров, скрин экрана DOM-тестера), справа — инфо-панель (sticky).
- Кнопки: «В корзину», «Купить в 1 клик», «В сравнение», «В избранное».
- Тех-таблица — datasheet-стиль: две колонки, mono для значений, числа по правому краю.
- DOM-блок — отдельная таблица (температура/напряжение/ток лазера/Tx/Rx Power/аварийные флаги) + поясняющий скрин консоли.
- Совместимость — раскрывающийся блок/вкладка с поиском (модуль → сотни моделей).
- «С этим покупают»: патч-корды, DAC, мультиплексоры.

## 7. Данные (модели / сущности / API)
Опирается на `Product` (K) + `Compatibility` (B).
- `GET /api/product/<sku>` → полный объект + firmwareOptions + documents.
- `GET /api/product/<sku>/compat?q=&page=` → пагинированная совместимость.
- `GET /api/product/<sku>/related` → аналоги (3 типа: та же скорость/др. дальность; тот же форм-фактор CWDM/DWDM; «с этим покупают»).
SEO JSON-LD: Product + Offer + brand + additionalProperty (Compatible Vendors) + BreadcrumbList (модуль J).

## 8. Доступность
Публично. Авторизованным B2B — индивидуальная/оптовая цена + «Запросить оптовую цену» для незарегистрированных.

## 9. Лимиты
- Таблица совместимости — поиск + пагинация (модуль → сотни моделей).
- Аналоги — по 4–8 на блок.

## 10. Этап реализации (тир)
- **P0:** галерея · основная инфо · тех-таблица · DOM-блок · таблица совместимости · выбор прошивки.
- **P1:** аналоги/«с этим покупают» · документы (PDF) · JSON-LD.
- **P2:** отзывы/оценки.

## 11. Приоритет
**P0** — после K/A/B.
