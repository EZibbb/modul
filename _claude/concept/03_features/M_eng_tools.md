# M. Инженерные инструменты — спецификация (11 полей)

> Доверие через глубину + инструменты для закупок. Опираются на K (каталог), G (калькулятор оптбюджета — отдельно), L (ИИ может их вызывать как tools).

## 1. ID + название
**M. Инженерные инструменты** — M1 декодер названия модуля · M2 калькулятор экономии (совместимый vs OEM) · M3 планировщик DWDM/CWDM · M4 конфигуратор линии А→Б.

## 2. Краткое описание
Набор утилит, которые делают сайт «умным справочником-калькулятором», а не просто магазином: разбор обозначений, расчёт выгоды, планирование спектра и подбор комплекта на линию.

## 3. Цель
Закрепить экспертный образ и дать закупкам/инженерам ощутимую пользу (особенно M2 — деньги). Эти инструменты дёшевы в сборке и сразу «вау».

## 4. Triggers / точки входа
- Раздел «Инструменты» в меню / база знаний.
- Контекстно: M1 — в поиске/карточке; M2 — в карточке и сравнении; M4 — из отраслевых лендингов.
- Как tools для ИИ-слоя (L вызывает M1/M2 при ответах).

## 5. Описание процесса
**M1 Декодер названия:** ввод `QSFP28-100G-LR4` → разбор по частям: форм-фактор (QSFP28), скорость (100G), стандарт (LR4 = одномод, 4 длины волны, 10 км). Парсер по правилам именования MSA/IEEE.
**M2 Калькулятор экономии:** выбор модуля + сравнение с ценой OEM (Cisco/Juniper) → «−68%, экономия 142 000 ₽ на партию N». Данные OEM-цен — в seed (ориентир).
**M3 Планировщик DWDM/CWDM:** интерактивная ITU-сетка (G.694.1/.2) → выбор каналов, визуальное заполнение спектра, подсветка конфликтов/занятых, подбор соответствующих модулей из каталога.
**M4 Конфигуратор линии А→Б:** ввод точек, расстояния, скорости, типа волокна → подбор комплекта (трансиверы + патч-корды + при необходимости мукс/усилитель) + расчёт бюджета (через G).

## 6. UI/UX
- Каждый инструмент — отдельная страница с чистым вводом и наглядным результатом.
- M1 — мгновенный разбор по мере ввода, плашки-пояснения терминов (глоссарий G).
- M2 — крупная цифра экономии + таблица, кнопка «в корзину».
- M3 — визуальная сетка спектра (горизонтальная шкала каналов).
- M4 — пошаговый ввод → итоговая спецификация + «добавить комплект».

## 7. Данные (модели / сущности / API)
- M1: правила парсинга (конфиг-таблица обозначений) — без БД-сущности.
- M2: `Product.priceBase` + `Product.oemPrice`/`oemRef` (seed-поле) → расчёт.
- M3: ITU-сетки (из K/G seed) + `Product` с `dwdmChannel`/`wavelengthNm`.
- M4: использует `search_products` + `get_compatibility` + `calc_optical_budget`.
- API: `GET /api/tools/decode?name=`, `POST /api/tools/savings`, `GET /api/tools/itu-grid`, `POST /api/tools/line-config`.

## 8. Доступность
Публично. M2 с индивидуальными ценами — для авторизованных B2B.

## 9. Лимиты
- M1 — покрывает основные MSA-обозначения seed-каталога (не все экзотические).
- M2 — OEM-цены ориентировочные (seed), помечать «для оценки».

## 10. Этап реализации (тир)
- **P1:** M1 декодер · M2 калькулятор экономии (дёшевы + сильный эффект для закупок).
- **P2:** M3 планировщик DWDM · M4 конфигуратор линии.

## 11. Приоритет
**P1** для M1/M2, **P2** для M3/M4. (ИИ-слой L может частично перекрывать M4 через L2 «подбор решения».)
