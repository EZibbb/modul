# E. Корзина и заказ — спецификация (11 полей)

> B2B-специфика: запрос КП, прошивка на позицию, быстрый заказ по Excel. Реальная оплата — roadmap (нет юрлица/платёжки до трудоустройства).

## 1. ID + название
**E. Корзина и заказ** — E1 корзина · E2 оформление заказа · E3 купить в 1 клик · E4 быстрый заказ по Excel.

## 2. Краткое описание
Корзина с B2B-фишками (комментарии, прошивка, запрос КП) и оформление для юр/физ-лиц; загрузка списка закупки из Excel.

## 3. Цель
Дать профессиональный B2B-флоу заказа и снять трение для повторных/объёмных закупок (Excel). Показать понимание, как закупают телеком-оборудование.

## 4. Triggers / точки входа
- «В корзину» / «Купить в 1 клик» из каталога, карточки, сравнения, ИИ-подбора, «добавить комплект» (B2/M4/L2).
- E4: страница «Быстрый заказ по списку» / в ЛК.

## 5. Описание процесса
**E1 Корзина:** список позиций, кол-во, удаление, промокод; на позицию — комментарий и выбор прошивки (firmwareOptions, K); расчёт с партнёрской скидкой; **прикрепить файл (ТЗ) → запрос КП** в отдел продаж; сохраняется для авторизованных.
**E2 Оформление:** юрлица (организация, ИНН, контактное лицо, оплата: счёт/карта), физлица/самозанятые (упрощённая форма), блок доставки (самовывоз/ТК/курьер), реквизиты, согласие → «Подтвердить заказ». **В демо:** заказ создаётся как заявка/КП (без реального эквайринга).
**E3 Купить в 1 клик:** мини-форма (контакт) → заявка.
**E4 Быстрый заказ по Excel:** скачать шаблон (артикул, кол-во, желаемый вендор, примечание) → загрузить → **валидация артикулов** (подсветка нераспознанных, предложение похожих) → перенос в корзину/заявку.

## 6. UI/UX
- Корзина — таблица позиций + сводка справа (sticky), поле промокода, кнопка «Запросить КП».
- Оформление — шаги/секции, переключатель юр/физ.
- E4 — drag&drop файла, таблица результата валидации с цветовой разметкой строк.

## 7. Данные (модели / сущности / API)
- **Cart**(id, userId?/sessionId, items[{productId, qty, firmware, note}], promo).
- **Order/Quote**(id, type[order/quote], customer{type, inn, org, contact}, delivery, items, attachments[], status).
- API: `POST /api/cart`, `POST /api/order`, `POST /api/quick-order/validate` (Excel → парсинг+матчинг по sku), `GET /api/quick-order/template`.
- Excel-парсинг (SheetJS), фаззи-матчинг артикулов.

## 8. Доступность
Корзина — все; сохранение между сессиями и индивид. цены — авторизованным B2B.

## 9. Лимиты
- Демо: оформление = создание заявки/КП, без реального платежа.
- E4: ограничение размера файла/числа строк.

## 10. Этап реализации (тир)
- **P1:** E1 корзина (с прошивкой/КП) · E4 быстрый заказ по Excel (сильный B2B-вау).
- **P2:** E2 полное оформление · E3 в 1 клик · реальная оплата (roadmap).

## 11. Приоритет
**P1** (E1, E4), **P2** (E2, E3, оплата).
