# 02 — Карта функционала + тиры

> Полный список функциональных модулей (A–N). Тиры P0/P1/P2 — предварительные (D-003), финализируем по ходу разбора. Детальные спеки каждой фичи — в `03_features/` по шаблону из 11 полей.

## Модули и фичи

| Модуль | Фичи |
|---|---|
| **A. Каталог и поиск** | Дерево категорий (форм-фактор/среда/применение) · фасетный фильтр · поиск (артикул/модель/характеристики + автодополнение) · сортировка |
| **B. Совместимость** 🔥 | Экспресс-виджет вендор→серия→модель · страница «Подбор совместимости» (по портам, комплект) · таблица совместимости в карточке · модель данных M2M товар↔модель |
| **C. Карточка товара** | Галерея + тех-таблица + DOM-блок + документы · выбор прошивки · аналоги / похожие / «с этим покупают» |
| **D. Сравнение** | Буфер сравнения + страница (до 4, подсветка различий) |
| **E. Корзина и заказ** | Корзина (сохранение, прошивка, запрос КП) · оформление (юр/физ, доставка, реквизиты) · купить в 1 клик · быстрый заказ по Excel |
| **F. Личный кабинет (B2B)** | Профиль/реквизиты · иерархические аккаунты · история заказов (повтор, счёт/УПД) · избранное и шаблоны · уведомления · оптовые/индивид. цены |
| **G. База знаний** | Калькулятор оптбюджета 🔥 · справочник длин волн CWDM/DWDM · глоссарий · блог/статьи |
| **H. Сервис и поддержка** | Программирование модулей (заявка) · тестирование на оборудовании · кастомизация |
| **I. Главная** | Композиция блоков (хедер, экспресс-подбор, категории, подборки, отраслевые, футер) |
| **J. Инфо / тех** | О компании · доставка/оплата · контакты · 404/поиск/карта сайта · SEO JSON-LD |
| **K. Данные (фундамент)** | Модель данных товара (атрибуты) · seed mock-каталог · матрица совместимости |
| **L. ИИ-слой** 🔥 | Тех-консультант (чат, RAG) · подбор решения по задаче · парсер конфига/ТЗ · диагностика DOM · генератор КП |
| **M. Инженерные инструменты** | Декодер названия модуля · калькулятор экономии (совместимый vs OEM) · планировщик DWDM/CWDM · конфигуратор линии А→Б |
| **N. UX-слой** | Cmd+K быстрый поиск · тёмная тема · сохранение/шаринг конфигураций |

## Тиры (предварительно — D-003)

### P0 — герой, работает идеально
K seed-данные (фундамент) · A каталог + фасеты · B кросс-фильтр совместимости · C карточка товара · **L: ИИ-консультант + подбор решения по задаче** · I главная · N: cmd+K + тёмная тема

### P1 — усиление, по готовности
L: парсер конфига + диагностика DOM · M: декодер + калькулятор экономии · G: калькулятор оптбюджета · D сравнение · E: корзина + Excel-заказ

### P2 — roadmap (слайд питча, не строим)
M: планировщик DWDM, конфигуратор линии · L: генератор КП · F: иерархич. аккаунты, оптовые цены · E: оплата/оформление · H сервис · J инфо/SEO · импорт 1С

## Зависимости
- **K (данные) — фундамент всего.** ИИ-фичи (L) и фасеты (A) держатся на богатом seed-каталоге + матрице совместимости. Делаем первым.
- **B (совместимость)** задаёт модель данных «товар ↔ модель оборудования» (M2M), от неё зависят виджеты, карточка, ИИ-подбор.
- **L (ИИ)** требует ключ Claude API + интернет → нужен запасной офлайн-режим на встрече (D-006).

## Порядок разбора (Этап 1)
1. **B. Совместимость** — killer + задаёт модель данных.
2. **K. Данные** — модель товара, атрибуты, матрица совместимости, seed.
3. **A. Каталог/поиск** → **C. Карточка** → **L. ИИ-слой** → далее по тирам.
