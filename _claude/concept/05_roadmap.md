# 05 — Roadmap: что в демо vs что в дорожной карте

> Чёткое разделение для питча: что реально работает в демо (P0/P1) и что показываем как направление роста (P2/боевое). Тиры сведены из `03_features/*`.

## 🎯 Стратегия демонстрации
На встрече: **«вот рабочий продукт (P0/P1), а вот ровно как он дорастает до боевого магазина (P2) при подключении к вашим системам».** Демо доказывает компетенцию, roadmap-слайд — что Евгений видит полную картину.

---

## ✅ P0 — Ядро демо (работает идеально, строим первым)
Фундамент + killer-фичи. Без этого демо не существует.

| Модуль | Что показываем |
|---|---|
| **K** Данные/seed | ~80–120 достоверных позиций + матрица совместимости |
| **B** Совместимость 🔥 | подбор вендор→модель, виджет, таблица в карточке |
| **A** Каталог | динамические фасеты + счётчики, поиск с автодополнением |
| **C** Карточка | datasheet, DOM-блок, таблица совместимости, выбор прошивки |
| **L** ИИ-слой 🔥 | тех-консультант (tool use) + подбор решения по задаче |
| **I** Главная | хедер, поиск, экспресс-подбор, категории |
| **N** UX | Cmd+K, тёмная тема |

## 🔶 P1 — Усиление демо (по готовности времени)
Добавляет глубину и «вау», но демо живёт и без них.

| Модуль | Что показываем |
|---|---|
| **L** ИИ | парсер конфига/ТЗ · диагностика DOM |
| **M** Инструменты | декодер названия · калькулятор экономии (для закупок) |
| **G** База знаний | калькулятор оптбюджета · ITU-сетки · глоссарий |
| **D** Сравнение | до 4 модулей, подсветка различий |
| **E** Корзина | корзина с прошивкой/КП · **быстрый заказ по Excel** |
| **F** Кабинет | профиль · история заказов (seed) · избранное |
| **N** UX | шаринг конфигураций ссылкой |
| **J** SEO | JSON-LD на карточках, 404 |

## 🗺️ P2 — Roadmap-слайд (НЕ строим, показываем направление)
Зависят от реальных систем/данных компании или избыточны для питча.

| Тема | Почему roadmap |
|---|---|
| **Реальная оплата** (эквайринг) | нужно юрлицо/платёжка компании |
| **Импорт из 1С/ERP** (CommerceML) | подключается тем же импортёром при доступе к 1С |
| **Иерархические корп-аккаунты** (F2) | права сотрудников — после реальной оргструктуры |
| **Индивидуальные цены из CRM** (F6) | зависит от прайсинга компании |
| **M3 планировщик DWDM · M4 конфигуратор линии** | глубокие инструменты, после ядра |
| **L5 генератор КП (PDF)** | приятно, но не критично для встречи |
| **H сервис** (формы заявок) | контент-доказательство, без реального воркфлоу |
| **J инфо-страницы, юр-часть** (оферта, 152-ФЗ) | нужны при реальном запуске |
| **Meilisearch/полнотекст** | при росте каталога за пределы seed |

## 🧭 Порядок сборки (Этап 2)
1. **Каркас:** Next.js + Tailwind + shadcn + Prisma + дизайн-токены (после D-005).
2. **K** — схема БД + детерминированный seed.
3. **B** — совместимость (модель + виджет + API).
4. **A** → **C** — каталог и карточка.
5. **L** — ИИ-слой (tool use на готовых сервисах).
6. **I** + **N** — главная + UX-лоск.
7. **P1** по готовности: G, M, D, E, F.
8. Деплой на Vercel + Neon, наполнение seed, прогон демо-сценариев.

## Параллельные треки
- **Дизайн** (D-005) — промт готов (`research/design_brief_prompt.md`), нужен ДО каркаса UI.
- **Офлайн-ИИ** (D-006) — предзаписать 2–3 showcase-сценария перед встречей.

## Definition of Done для демо
- P0 полностью работает на seed-данных, задеплоено, открывается по ссылке.
- 2–3 отрепетированных ИИ-сценария + офлайн-фолбэк.
- Roadmap-слайд (этот документ) — для разговора о росте в боевое.
