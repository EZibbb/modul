## @section: CONCEPT

**«Инженерная точность как интерфейс».** Приборная панель, а не витрина гаджетов. Данные — герой; декор минимизирован; палитра спокойная и отдаёт пиксели таблицам и числам.

**5 принципов:** инженерная точность · плотность без хаоса · технологичность (не игрушечность) · доверие (бейджи «протестировано», статусы, сертификаты) · адаптивность (десктоп-first, мобильные фильтры в offcanvas, тач ≥ 44px).

**Аудитория-обоснование:** инженеры связи / сетевые админы / проектировщики / закупки ценят плотные читаемые datasheet-таблицы, моноширинные числа, быстрый параметрический подбор, тёмную тему. Позиционирование: современнее РФ-конкурентов (NAG/FIBO/BAZIS/Ланцет), к эталонам FS.com / Linear / Vercel / Stripe.

## @section: TOKENS

> Источник истины — `design/assets/tokens.css` (HSL-каналы, формат shadcn: `:root` + `.dark`).

**Бренд:**
| Токен | HEX (light) | Назначение |
|---|---|---|
| `--primary` | `#0A4FD6` | Глубокий оптический синий — действия, фокус |
| `--cyan` | `#06B6D4` | Фотонный циан — акцент в данных |

**Темы — равноправные** (не инверсия):
| | Light | Dark |
|---|---|---|
| `--background` | `#F4F6FA` (холодный — белые карточки «всплывают») | `#0B0F17` (slate-950, «приборная» среда) |
| `--card` | `#FFFFFF` | `#10151F` |
| `--primary` | `#0A4FD6` | `#3B82F6` (поднят для тёмного фона) |

**Семантика:** success (в наличии/протестировано) · warning (под заказ) · destructive (EOL) · info (DOM/в пути). Data-viz ramp `--data-1…5` (синий/циан/зелёный/янтарь/фиолетовый) для оптбюджета и диаграмм.

**Типографика:** **Inter** (UI/текст, 400/500/600/700) + **IBM Plex Mono** (артикулы, λ, °C, DOM, числа — `tabular-nums`). H1 36px → Body 15px → Caption 13px → Mono 13px. Доп. размер `text-2xs` = 11px.

**Геометрия:** радиус `--radius: 0.5rem` (8px); сетка на базе 4px (4·8·12·16·24·32·48); контейнер max 1320px, поля 24px. Тени `--shadow-xs…lg` (мягкие).

## @section: OPTICAL_LANGUAGE

> `design/assets/base.css` — фирменный приём, отличает от «обычного shadcn».

| Класс | Что |
|---|---|
| `.hero` / `.page-head` | Тёмная фотонная поверхность (dark) / тема-зависимая шапка (светлая «приборная» в light) |
| `.fiber-line` + `.pulse` | Горизонтальная «волоконная» линия с движущимся световым импульсом (анимация `travel`) |
| `.spectrum-rule` | Спектральный разделитель (градиент длин волн) |
| `.grid-bg` / `.hero-grid` | Инженерная сетка с радиальной маской |
| `.glass` / `.glass-field` | Стеклянные панели (dark) → белые карточки (light) |

Мотив длин волн **850 / 1310 / 1550 нм** (eyebrow, шкала). Все анимации гаснут при `prefers-reduced-motion: reduce`.

## @section: INTEGRATION

**Стек-нюанс:** дизайн собран на **Tailwind 3 CDN** (`design/assets/tw-setup.js` = объект конфига), наш `web/` — **Tailwind 4** (`@theme` в CSS вместо `tailwind.config.ts`). CSS-переменные совместимы 1:1 → токены переносятся прямо; маппинг утилит-цветов адаптируется под v4.

**План переноса (roadmap 2.3):**
1. `tokens.css` → `web/src/app/globals.css` (`:root` + `.dark` + `@theme`).
2. `base.css` (оптический язык, scrollbars, focus) → `globals.css`.
3. Шрифты Geist → Inter + IBM Plex Mono через `next/font/google`; анти-FOUC тёмной темы (W11).
4. Компоненты — на shadcn/ui (маппинг Modul→shadcn в `design/SPEC.md` §8.3).
5. JS-логика экранов (`design/assets/*.js`) → React-состояние при сборке экранов (2.4).

**Соответствие shadcn** (SPEC §8.3): Button→button · Badge→badge · ⌘K→command+dialog · Tabs→tabs · ProductCard→card · DomTable→table · FirmwareSelect→select · ИИ-drawer→sheet · модалки→dialog · тосты→sonner.
