# Backups · Decisions snapshots

> Здесь хранятся снапшоты `_claude/` (и опционально других папок проекта) после **ключевых этапов**.

---

## Когда делать бэкап

См. правило в `_claude/INSTRUCTIONS.md` → секция «💾 БЭКАП — КОГДА ДЕЛАЕМ И КОГДА НЕТ».

Кратко:
- ✅ После финализации **принципиального документа** (concept/overview, architecture, monetization, roadmap)
- ✅ После **завершения блока** из active.md (закрыта подсистема / этап)
- ✅ Перед **опасным действием** (массовое удаление / перезапись большого числа файлов)
- ❌ НЕ делаем для обычных правок и рутины

**Правило одного бэкапа в сессию:** максимум 1 бэкап на финал важного блока. Повторные — НЕ нужны.

---

## Формат

```
backups/decisions/YYYY-MM-DD_короткое_описание/
├── _claude/             ← снапшот мозга проекта на эту дату
└── _АКТУАЛЬНЫЙ.md       ← обязательный файл-сопроводилка
```

В `_АКТУАЛЬНЫЙ.md`:
- Что именно фиксируем (этап / документ / событие)
- Какие D-решения закрыты
- Что изменилось vs прошлый бэкап
- Что в следующем шаге

---

## Пример имени

- `2026-05-19_etap2_close/` — закрытие Этапа 2
- `2026-05-18_design_v1_v2/` — получены дизайны v1+v2
- `2026-05-16_etap1_complete/` — финал Этапа 1 (концепция)

---

## Восстановление из бэкапа

```powershell
# Просто скопировать обратно нужный снапшот
Copy-Item -Path "backups/decisions/YYYY-MM-DD_xxx/_claude" -Destination "." -Recurse -Force
```
